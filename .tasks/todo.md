- 滚动条实现原理
```
1. 根据单元格数量，计算整体内容高度/宽度
2. 根据绘制区域的视口高度(container)，与整体内容的高度，计算比例 ratio => 也就是当前视口高度，占整体高度的比例
3. 滑块的高度/宽度 = 视口高度(container) * ratio

  计算逻辑：
  - 滑块长度 = 视口大小 × (视口/内容)，最小20px
  - 滑块位置 = (当前滚动位置/可滚动范围) × 滑块可移动范围

主要原理（比率映射）:
1. 视口高度与实际高度的占比 * 视口高度 = 滑块高度  （比例 * 参照物） 滚动条轨道 ↔ 滑块位置
2. 滑块位置 = 当前滚动位置 / 可滚动高度 * 滚动条最大可移动位置 （比例 * 参照物） 块位置 ↔ 内容滚动位置

```
```
1. scrollableSize 计算

  const scrollableSize = this.contentSize - this.viewportSize

  含义： 计算可滚动的内容大小

  例子：
  - 内容总高度 contentSize = 1000px
  - 视口高度 viewportSize = 300px
  - 可滚动大小 scrollableSize = 1000 - 300 = 700px

  这表示内容可以向上滚动700px才能完全显示。

  2. thumbPosition 计算

  const thumbPosition = scrollableSize > 0
    ? (this.scrollPosition / scrollableSize) * maxThumbPosition
    : 0

  当 scrollableSize > 0 时（内容超出视口）：

  公式： 滑块位置 = (当前滚动位置 / 可滚动大小) × 滑块最大移动距离

  具体计算过程：

  假设：
  - scrollPosition = 350px （当前滚动了350px）
  - scrollableSize = 700px （总共可滚动700px）
  - maxThumbPosition = 200px （滑块最多可移动200px）

  计算：
  滚动进度百分比 = 350 / 700 = 0.5 (50%)
  滑块位置 = 0.5 × 200 = 100px

  意义： 当内容滚动了50%时，滑块也应该移动到滑轨的50%位置。

  当 scrollableSize <= 0 时（内容未超出视口）：

  thumbPosition = 0

  意义： 如果内容完全可见（不需要滚动），滑块始终保持在起始位置。

  3. 比例关系图解

  内容滚动     ←→     滑块移动
     0px       ←→        0px    (顶部)
    350px      ←→      100px    (中间)
    700px      ←→      200px    (底部)

  这种比例映射确保了：
  - 滑块位置始终反映内容的滚动进度
  - 拖拽滑块时能精确控制内容位置
  - 避免了除零错误和边界问题

  这就是自定义滚动条实现"所见即所得"滚动体验的数学原理。
```
```
滚动条位置计算原理

比例映射滚动的原理，将滚动条UI操作转换为内容滚动：
  核心原理

  双重映射关系：
  - 滚动条轨道 ↔ 滑块位置
  - 滑块位置 ↔ 内容滚动位置

  计算步骤

  1. 坐标转换：将鼠标点击的屏幕坐标转换为相对于滚动条轨道的局部坐标
  2. 中心对齐：让滑块中心对准点击位置，而不是滑块边缘，提供更直观的交互体验
  3. 比例计算：
  比例 = 滑块位置 / 滑块可移动范围
  4. 内容映射：
  滚动距离 = 比例 × 内容可滚动范围

  数学模型

  - 滑块可移动范围 = 轨道长度 - 滑块长度
  - 内容可滚动范围 = 内容总大小 - 视口大小
  - 映射函数：f(滑块位置) = 内容滚动位置

  这样无论内容多大、滚动条多小，都能通过比例关系准确定位到内容的对应位置。
```
